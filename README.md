# АЗС Ставрополь — бот

## 1. Получить токен бота
1. В Telegram открой **@BotFather**
2. `/newbot` → придумай имя и username (должен заканчиваться на `bot`)
3. Скопируй токен вида `123456:ABC-DEF...`

## 2. Подключить приём платежей (для /subscribe)
1. В том же чате с **@BotFather**: `/mybots` → выбери бота → **Payments**
2. Выбери провайдера, работающего в России — например **ЮKassa** (ЮMoney)
3. Пройди привязку своего аккаунта ЮKassa (нужно ИП или самозанятость для вывода денег)
4. BotFather выдаст **Payment Provider Token** — его нужно вставить в переменную `PAYMENT_PROVIDER_TOKEN`

Без этого шага бот запустится и будет работать, но `/subscribe` просто скажет, что оплата не подключена — остальные команды не пострадают.

## 3. Запуск локально (для проверки)
```bash
pip install -r requirements.txt
export BOT_TOKEN="твой_токен"
export PAYMENT_PROVIDER_TOKEN="токен_от_юkassa"   # можно пропустить, если оплату пока не настраивал
python bot.py
```
Открой бота в Telegram, напиши `/start`.

## 4. Бесплатный деплой на Railway / Amvera (чтобы работал 24/7)
1. Зарегистрируйся (Railway — через GitHub; Amvera/TimeWeb — через российскую карту, если Railway не заведётся)
2. Залей `bot.py`, `requirements.txt` в GitHub-репозиторий
3. New Project → Deploy from GitHub repo → выбери репозиторий
4. В Variables добавь `BOT_TOKEN` и `PAYMENT_PROVIDER_TOKEN`
5. Платформа сама поставит зависимости и запустит `python bot.py`

База данных (`fuel.db`) создаётся автоматически при первом запуске рядом с ботом.

## Команды бота
- `/start` — приветствие и список команд
- `/report` — сообщить статус на АЗС (станция → статус → топливо → комментарий)
- `/list` — текущая обстановка по всем станциям
- `/addstation` — добавить новую АЗС
- `/subscribe` — платная подписка (149 ₽ / 30 дней) на уведомление, когда на выбранной АЗС появится бензин
- `/mysubs` — список активных подписок пользователя

## Как работает уведомление
Когда кто-то отправляет `/report` и статус станции меняется с "нет/очередь" на "есть/мало" — бот
автоматически рассылает сообщение всем, у кого активна подписка на эту станцию. Проверка встроена
прямо в обработку отчёта, отдельный воркер не нужен.

## Дальше
- Изменить цену/срок подписки — переменные `SUB_PRICE_RUB` и `SUB_DAYS` в начале `bot.py`
- Когда будет 20-30 активных пользователей — можно перенести базу на Supabase, чтобы объединить с веб-версией
- Локальная реклама: добавить команду вроде `/sponsor`, которая раз в день добавляет в `/list` короткую подпись от партнёра
